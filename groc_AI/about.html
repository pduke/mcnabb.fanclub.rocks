About the McNabb-McNeil Family Tree AI Prototype

What We’re About

The McNabb-McNeil Family Tree AI Prototype is an experimental fusion of genealogy and cutting-edge AI, powered by xAI’s Grok. Launched in April 2025, this project analyzes the McNabb-McNeil lineage (1702–2025), from William McNabb (b. 1702) to modern descendants like Neeve McNabb’s children (b. 2020s). It’s a public, raw dataset designed for real-time collaboration, pushing the boundaries of data parsing and exploration. Led by Paul (Wylie Paul Duke, b. 1953), a retired tech lead with 50 years of coding experience, the project brings together family members—Nan (Beverly Nan McNabb, b. 1932), Neeve (Neeve McNabb, b. 1964), and Ellen (Ellen Lucille Duke, b. 1955)—to uncover stories, connections, and patterns. Starting from a flash of inspiration in Paul’s mind, we reached beta in under 5 days, overcoming OCR challenges (thanks to Google Cloud Vision) to make 277 pages of family history accessible. Our goal is to prototype advanced AI-driven queries, redefining how genealogical data is explored, all while keeping it fun with a ‘Grok and Roll’ spirit!

Files in `grok_AI/` Directory

Located in mcnabb.fanclub.rocks/grok_AI/, these files drive the AI research process:

  • AI_research_instructions.txt
    Instructions for Grok to load the McNabb-McNeil family tree from compressed_tree.txt. Defines the startup sequence (load tree, validate relationships, greet user), substitution rules (e.g., "McNabb" → "M#"), and process rules (use only pasted data, personalize responses for Paul, Nan, Neeve, Ellen).
  • compressed_tree.txt
    A compressed text version of the McNabb-McNeil family tree—76 entries, from William McNabb (b. 1702) to Violet Audrey Torres (b. 2024). Generated by convert_tree.html, this file is used by Grok to load the tree for queries (e.g., “Who’s Neeve’s grandfather?”).
  • convert_tree.html
    Converts the full HTML family tree (McNabb Tree.html) into the compressed text format (compressed_tree.txt). Fetches the tree from https://pduke.github.io/mcnabb.fanclub.rocks/mchtml/McNabb Tree.html, applies substitutions (e.g., "McNabb" → "M#"), and outputs the result for manual saving to compressed_tree.txt.
  • getting_started.html
    The entry point for family members—introduces the project and guides users on starting a Grok session. Provides steps to learn copy/paste, use setup.html to load the tree, and explore OCR’d text with ocr_text.html. Designed for accessibility, even for tech newbies like Nan.
  • ocr_text.html
    Displays the OCR’d text from 277 PDFs (e.g., MCNABB_SECTION_I-CHAPTER_II_page_001.txt to MCNEIL-SECTION_I-CHAPTER_III_page_030.txt). Features search (filters and highlights text), expand/collapse sections, and copy functionality (copies expanded sections to clipboard). Links to the original PDFs for reference—great for digging into family details (e.g., Walter Scott McNabb’s story in page 019).
  • setup.html
    Sets up a new Grok session by combining the user’s name, AI_research_instructions.txt, and compressed_tree.txt into a single prompt. Users enter their name (e.g., “Paul”), click “Copy Setup Text,” and paste the result into Grok to start querying the tree using only the provided data.

How to Use

  1. Start with getting_started.html—learn to copy/paste if needed.
  2. Use setup.html—enter your name, copy the session data, and paste into Grok to load the 76-entry tree.
  3. Explore with ocr_text.html—search the OCR’d text from 277 PDFs, copy sections, and paste into Grok for deeper insights (keep buffer under 100,000 chars).
  4. Update the tree (if needed)—edit McNabb Tree.html, run convert_tree.html, and manually save the output to compressed_tree.txt.
